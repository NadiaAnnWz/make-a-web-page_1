Stage 0

Introduction

-The web is used to access information by way of computers.
-The user types a URL into their browser, or searches for one, which results in the computer communicating with servers by way of HTTP and the Internet.
-The accessed information is in the form of webpages that contain HTML (hypertext markup language).

Aspects of HTML

-Text: visible
-Markup: how it is arranged
-References to other documents: images and videos
-Links to other pages.

Tags

-What markup is made of: opening and closing tags
-Example: <tag name> content </tag name>
-A basic tag would be one to make text bold: <b> text </b>

Tag Attributes

-Used for images (src) and links (href)
-Images do not have closing tags. Here is an example of an image using an src attribute:
-The image tag would be written as: <img src=http://4.bp.blogspot.com/-QJfGXFeAR1w/T9xVKyzb3kI/AAAAAAAAAFM/m7MINdzgzLo/s1600/blue+point+siamese.jpg alt=siamese!>
-Example of a link using a href attribute: Cat Fancy!

Whitespace and Line Breaks

-To create multiple lines, the tags used are for Breaks (br) and Paragraph (p)
-Breaks are considered a 'Void Tag' meaning they do not have a closing tag.
-Paragraphs require a closing tag. Paragraph tags essentially create a 'box' in which the text is written, of which we will eventually learn to format the size.

Inline vs. Block Elements

-Inline elements results in changes to a line of text. Tags include attribute, span, break, image, and strong.
-Block elements format entire blocks of information. They include div, form, and paragraph.

Stage 1

Programming Languages and Purposes

-HTML creates the structure of a website
-CSS provides the styling of a website
-JAVA, which is not learned in this course, provides interactive elements to the website

Viewing Developer Tools

-Provides ability to see each level of HTML/CSS elements
-Items like fonts, border sizes, colors, etc. are visible
-The Document Object Model (DOM) tree is the structure of the page

HTML/CSS Syntax and Rules in Text

-Tags turn into elements that form the DOM tree
-In HTML, content is indicated between open and close tags
-In CSS, content indicated will change colors, fonts, etc.

Everything is a Box

-Every element should have a box around it, whether visible or not
-Using HTML and CSS coding, aspects of the boxes can be identified, including size, color, and content
-Boxes are typically indentified early in the process by sketching on paper, or cutting them out

Class Attributes

-Each box should be identified with a div tag containing the following: <div class = identifier> content </div>
-By identifying each division at each level, the boxes can be formatted to suit the webpage
-This ultimately creates the DOM tree

CSS - Cascading Style Sheets

-Cascading: descending rules of each element
-Sheets: there can be several; provide visible style to the webpage
-Strictest rules are applied to every element

Avoiding Repetition

-Avoiding repetition is important to avoid errors
-Minor changes, inconsistently applied, can result in problems on a webpage.
-CSS allows style to be applied to a large quantity of HTML code, without the need for repetition by assigning attributes to different items in the same class.

Inheritance

-Ancestor/descendant relationship, relies on a DOM tree or hierarchy of HTML elements
-Color, font, spacing, line height, list style, text alignment, etc. can all be inherited.

Aspects of CSS

-Selector: the elements to which the style will apply. Contains an attribute and a value. For example, a paragraph with a black background color and white writing would be written as: p {background-color : black; color: white; }
-In that example, 'p' is the selector, "background-color: black" is a declaration, "background-color" is a property, and "black" is the value.
-Tags: identify where on the page you are. For example, "h1" is the title. Div tags become h1, h2, etc.
-The above-mentioned headings have default settings that automatically change the font and spacing aspects.
-Add at the beginning of your style CSS to assure appropriate view in different browsers:
*{ -webkit-box-sizing: border-box; -moz-box-sizing: border-box; -ms-box-sizing: border-box; box-sizing: border-box;}

Box Model

-Content: center of the box, containing image and text shown on the website.
-Padding: clears area around the content; affected by the background color.
-Border: area surrounding padding and content; inherited style from color property.
-Margin: clears area around border; contains no color.
-The above-mentioned items can be adjusted for extra spacing.
-An entire element includes all four of these aspects.
-Use the MDN CSS list to identify width, height, margin width, etc.

Sizing and Positioning

-Percentage Sizing: size of box will change based on size of screen. Allows for nicer viewing experience if screen size is adjusted.
-The box sizes change but the layout remains the same.
-Flexbox is helpful when adding images to the webpage.

Testing and Refining

-Structure: "boxify" distinct boxes within content.
-Tags: use to make headings
-Handle minor details last.
